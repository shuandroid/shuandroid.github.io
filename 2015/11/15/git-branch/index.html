git branch 多分支

一些与他人合作的 git 多分支经验

创建分支chen 、dan 还有一个默认的master分支。

  • 在chen上做一次修改：
    git 命令：
    • git add . //先add
    • git commit -m “test from chen”
    • git checkout master //切换到分支master
    • git pull origin master //先从master拉下来东西，以防分支dan 已经向master提交东西
    • git checkout chen // 重新回到分支chen
    • git merge master //merge master 上的的代码
    • git push origin chen //正式提交 分支chen 上的修改代码
    • git checkout master //切换到master分支
    • git merge chen //同步分支chen 上的值
    • git push origin master //提交master 上总的修改（包括分支chen , dan 上的修改）。
    • git checkout chen // 重新回到chen 分支
